The app consist of a number of classes/modules that each have its limited responsibilities. The idea is that they are decoupled from each other so each module can be used in any other implementation. They are all singletons which mean there is only one instance of each module.
The scripts have not been minified to retain the readability of the code.

  • App is responsible for handling interactions and events in the document. This is the main class that is the connection between the html document and the other modules.
  • Facebook is responsible for all interactions with the Graph API, including; loading the JavaScript SDK, handling authorisation, response errors and caching. It's the bridge between the app and the facebook api.
  • Store is responsible for caching data, in the session storage if available, or otherwise in an alternative storage object. This is used for caching facebook api response data as well as calculated word data.
  • Progress is responsible for showing progress indication on the page. It's through this module that proress is started, updated and ended in a controlled way.
  • Words is responsible for handling and storing words. It's the interface through where the calculated data (the words) is available.
  • Logger is responsible for logging on levels Debug, Timer, Warning and Error. I use this module to notify the user of any restrictions or errors that might happen and also for debugging when developing.
  • Resize is responsible for adjusting the size of a specified element when the window is resized.
  • Extract Words Worker contains the code used to extract words, which is used both by asynchronous workers for browsers where Web Workers are supported and for the synchronous implemnetation. The script file also represent an actual web worker.
  • Word Cloud is responsible for the initialisation and rendering of the word cloud visualisation.
  • Bar Chart is responsible for the initialisation and rendering of the bar chart visualisation.
